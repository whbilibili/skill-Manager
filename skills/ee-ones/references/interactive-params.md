# 写操作参数清单与交互流程

> **加载时机**：当 Agent 需要执行**创建、更新、删除**等写操作，需要确认必填参数、交互流程或字段取值范围时，读取本文件。

---

## 创建工作项（wc）

**交互流程**：选择空间 → 选择子类型 → 填写必填字段 → 确认预览 → 提交

| 参数 | 说明 | 必填 |
|------|------|------|
| `-t, --type` | 类型: REQUIREMENT / DEVTASK / DEFECT（支持中文：需求/任务/缺陷/bug） | 是 |
| `-p, --project-id` | 空间 ID（跳过空间选择） | 推荐 |
| `-s, --space-name` | 空间名称关键字 | 替代 -p |
| `--title` | 标题 | 推荐预填 |
| `--desc` | 描述 | 可选 |
| `--priority` | 优先级：低/中/高/紧急 或 1~4 | 可选 |
| `--severity` | 严重程度（缺陷专属）: BLOCKER/CRITICAL/MAJOR/NORMAL/TRIVIAL | 缺陷时可选 |
| `--assigned` | 指派给（MIS 号） | 可选 |

> **自动化建议**：尽量提供 `-p`、`--title`、`--priority`，减少交互步骤。

---

## 更新工作项（wu）

| 参数 | 说明 |
|------|------|
| `-i, --id` | 工作项 ID |
| `--space` | 空间 ID（可选，自动提取） |
| `-f, --field` | 字段名（模糊匹配） |
| `--val` | 新值（跳过交互选择） |
| `-d, --desc` | 自然语言描述更新内容（AI 推断字段） |

**可修改字段**：标题、优先级、状态、迭代、子类型、指派给、预计开发开始/结束时间、预计开始/结束时间、预计上线时间

**注意**：
- 子类型只能更新为**同类型**下的合法子类型（如需求只能改成需求类子类型）
- `-d` 模式依赖 AI 推断，不确定时用 `-f` + `--val` 显式指定

---

## 删除工作项（wd）

| 参数 | 说明 |
|------|------|
| `-i, --id` | 工作项 ID |
| `-c, --delete-children` | 同时删除子工作项 |
| `-y, --yes` | 跳过确认直接删除 |

> ⚠️ 删除不可恢复，agent 执行前应先 `workitem-detail` 确认。

---

## 创建迭代（ci）

**交互流程**：输入空间ID → 名称 → 日期范围 → 描述 → 人力配置 → 确认

| 参数 | 说明 |
|------|------|
| `-p` | 空间 ID |
| `-n` | 迭代名称 |
| `-s` | 开始日期（YYYY-MM-DD） |
| `-e` | 结束日期（YYYY-MM-DD） |
| `-d` | 描述 |
| `--hr` | 人力配置: last（默认）/ project |
| `-y` | 跳过确认 |

> 自动化时传入全参数 + `-y`。

## 创建提测单（stc）

**交互流程**：选择空间 → 选择子类型 → 关联工作项（必填，支持多选，可关联分支）→ 填写必填字段 → 确认

| 参数 | 说明 |
|------|------|
| `-p` | 空间 ID |
| `-s` | 空间名称关键字 |
| `--title` | 标题 |
| `-y` | 跳过确认 |

> 创建提测单时必须关联至少一个工作项。

---

## 更新提测状态（stu）

**交互流程**：查询详情 → 获取可流转状态列表 → 选择目标状态 → 检查必填字段 → 执行

| 参数 | 说明 |
|------|------|
| `-p` | 空间 ID（必填） |
| `-i` | 提测单 ID（必填） |
| `-s` | 目标状态名称（模糊匹配） |
| `-y` | 跳过确认 |

> 若当前提测单已是终态或无权限，则无可流转选项。

---

## 分支创建与关联

### 创建分支（bc）

**流程**：查询工作项详情 → 生成分支名 → 搜索空间应用（未指定 -a 则交互选择）→ 创建并关联

| 参数 | 说明 |
|------|------|
| `-i` | 工作项 ID（必填） |
| `-t` | 分支类型（默认 feature） |
| `-a` | 应用 ID（不传则交互选择，通过 `apps` 命令获取） |
| `-b` | 基础分支（默认 master） |

### 关联已有分支（ba）

| 参数 | 说明 |
|------|------|
| `-n` | 分支名称（必填） |
| `-p` | 空间 ID（必填） |
| `-a` | 应用 ID（必填） |
| `-t` | 工作项 ID（必填） |
| `--branch-type` | 分支类型（默认 feature） |

> 关联前确保分支已推送到远程仓库。

## 工时填写（wtadd）

| 参数 | 说明 | 必填 |
|------|------|------|
| `-i` | 工作项 ID | 是 |
| `--hours` | 工时数：8h / 1d / 纯数字 | 是 |
| `-d` | 日期，默认今天；范围: YYYY-MM-DD~YYYY-MM-DD | 可选 |
| `--add-type` | 填写类型，默认 TOTAL | 可选 |
| `--desc` | 工时描述 | 可选 |
| `--include-weekend` | 包含周末（默认 false） | 可选 |
| `-y` | 跳过确认（周末默认不包含） | 自动化时推荐 |

> 提交前会自动执行参数校验（preValidations），校验不通过会阻断提交。

---

| 参数 | 说明 | 必填 |

| `-i` | 工作项 ID | 是 |
| `--hours` | 新的工时数 | 可选 |
| `-d` | 日期 | 可选 |
| `--desc` | 描述 | 可选 |

---

## 工时删除（wtdel）

| 参数 | 说明 |
|------|------|
| `--id` | 工时记录 ID（通过 `wtd` 获取） |
| `-y` | 跳过确认 |

> ⚠️ 工时记录 ID 必须通过 `ones wtd -i <工作项ID>` 查询获取。
