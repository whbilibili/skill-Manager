# 写操作参数清单与交互流程

> **加载时机**：当 Agent 需要执行**创建、更新、删除**等写操作，需要确认必填参数、交互流程或字段取值范围时，读取本文件。

---

## 创建工作项（wc）

**交互流程**：选择空间 → **⛔ 空间权限前置校验** → 选择子类型 → 填写必填字段 → 确认预览 → 提交

> ⚠️ **空间权限前置校验**：`wc` 命令在参数解析后会**立即校验**用户是否有目标空间的权限（`assertSpacePermission`），无权限时直接阻断退出，不会继续执行后续的字段校验、界面方案查询等重操作。这避免了用户填了大量参数后才发现无权限的糟糕体验。

| 参数 | 说明 | 必填 |
|------|------|------|
| `-t, --type` | 类型: REQUIREMENT / DEVTASK / DEFECT（支持中文：需求/任务/缺陷/bug） | 是 |
| `-p, --project-id` | 空间 ID（跳过空间选择） | 推荐 |
| `-s, --space-name` | 空间名称关键字 | 替代 -p |
| `--title` | 标题 | 推荐预填 |
| `--desc` | 描述 | 可选 |
| `--priority` | 优先级：低/中/高/紧急 或 1~4 | 可选 |
| `--severity` | 严重程度（缺陷专属）: BLOCKER/CRITICAL/MAJOR/NORMAL/TRIVIAL | 缺陷时可选 |
| `--assigned` | 指派给（MIS 号） | 可选 |

> **自动化建议**：尽量提供 `-p`、`--title`、`--priority`，减少交互步骤。

### ⚠️ 字段取值规则（创建 & 更新通用）

以下规则适用于 `wc`（创建）和 `wu`（更新）中通过 `-F / --field` 传入的自定义字段：

**1. 普通下拉字段（list / component_* 等）**

所有下拉接口返回的结构都是 `{ value, displayValue }`，**实际填值必须使用 `value`**，不能用 `displayValue`。

```bash
# 先查可选值
ones fo -p <空间ID> -f <字段ID> -t <字段类型>
# 返回示例: [{ "value": 42, "displayValue": "高优方案" }, ...]
# 传入 fieldValue 时使用 value（42），不是 displayValue（"高优方案"）
```

**2. 级联字段（cascader 类型）**

级联字段有多层级可选值（children 嵌套），**只允许选择最末级节点**（无 children 或 children 为空的节点）。

传值规则：**各层级的 value 用 `^` 拼接**，从第一层到最末层。

```bash
# 假设级联结构：
#   产品线A (value: "1")
#     └─ 子模块X (value: "2")
#           └─ 功能点Y (value: "3")   ← 最末级
#
# 传值：fieldValue = "1^2^3"
-F '{"id":1234,"name":"所属产品","variable":"customField1234","type":"cascader","multiple":false,"fieldValue":"1^2^3"}'
```

> ⚠️ **严禁选择非末级节点**。`^` 是唯一合法的层级分隔符，不能用 `/` 或其他字符。

**3. 需求提出部门字段（component_new_org 类型，如 customField23388）**

此类型字段的值是**组织 ID（数字）**，不是组织名称。获取组织 ID 的方式：

- 用户提供**完整部门链**（如 `美团/核心本地商业/基础研发平台/研发质量与效率部`）
  > 💡 完整部门链可从**大象个人名片**中查找（如：美团/行政/行政BP中心）
- 通过接口查询：`GET /api/proxy/org/getOrgByPath?path=<URL编码路径>`
- 取返回值中的 `id` 作为 `fieldValue`

```bash
# 示例：用户提供路径 "美团/核心本地商业/基础研发平台/研发质量与效率部"
# 接口返回 { "id": 40000923, "name": "研发质量与效率部", ... }
# 传值：fieldValue = 40000923
-F '{"id":23388,"name":"需求提出部门","variable":"customField23388","type":"component_new_org","multiple":false,"fieldValue":40000923}'
```

> Agent 在遇到 `component_new_org` 或 `component_org` 类型必填字段时，应提示用户提供完整部门链（可从**大象个人名片**中查找，如：美团/行政/行政BP中心），然后调用上述接口获取 ID。

## 更新工作项（wu）

> ⚠️ **空间权限前置校验**：`wu` 命令在解析工作项 ID 后，会**立即查询工作项详情**获取所属空间并校验权限（`getIssueDetail` 内含 `assertSpacePermission`），无权限时直接阻断退出，不会进入级联字段校验、预计工作量换算等重操作。

| 参数 | 说明 | 必填 |
|------|------|------|
| `-i, --id` | 工作项 ID | 是 |
| `--name` | 工作项标题 | 可选 |
| `--desc` | 描述信息（支持 HTML） | 可选 |
| `--priority` | 优先级: 低(1) / 中(2) / 高(3) / 紧急(4) | 可选 |
| `--assigned` | 指派给（用户 MIS 号） | 可选 |
| `--expect-time` | 预计工作量（8h / 1d / 0.5d / 纯数字），自动根据空间工时配置换算 | 可选 |
| `-F, --field` | 自定义字段（WorkitemValueField JSON，可多次使用） | 可选 |
| `-y, --yes` | 跳过确认直接更新 | 自动化时推荐 |

**注意**：
- 至少提供一个更新字段（`--name` / `--desc` / `--priority` / `--assigned` / `--expect-time` / `-F`）
- 预计工作量会自动获取工作项详情确定空间 ID，再根据空间工时配置换算单位
- `-F` 参数接收 JSON 格式：`{"variable":"...","name":"...","type":"...","multiple":false,"fieldValue":...}`
- 查询字段元信息：`ones fs -p <空间ID> -t <工作项类型>`；查询可选值：`ones fo -p <空间ID> -f <字段ID>`

## 创建大象群（wxg create）

**交互流程**：检查已关联 → 查询字段 → 查询成员 → 展示预览 → 调用接口创建

| 参数 | 说明 | 必填 |
|------|------|------|
| `-i, --issue-id` | 工作项 ID | 是 |
| `-t, --issue-type` | 工作项类型: REQUIREMENT / DEVTASK / DEFECT / PRODUCT_DOCUMENT | 是 |
| `--users` | 手动指定成员 MIS（逗号分隔），不传则自动查询 | 可选 |
| `--admins` | 群管理员 MIS（逗号分隔，默认取成员列表中首个） | 可选 |
| `--owner` | 群主 MIS（默认取成员列表中首个） | 可选 |
| `--user-fields` | 群名/备注中展示的字段名称（逗号分隔），默认取默认勾选字段名 | 可选 |
| `--include-children` | 是否包含子工作项成员（默认 false） | 可选 |
| `--json` | 以 JSON 格式输出创建结果 | 可选 |

> **注意**：
> - 该命令含**幂等检查**：若工作项已关联大象群，则直接展示已关联群聊信息并退出，不会重复创建
> - 不传 `--users` 时，自动按以下顺序获取成员：① 查询可选字段定义（roles 接口）→ ② 筛选 `isSelected=true` 的字段 → ③ 查询这些字段对应的人员 MIS（members 接口）
> - 若自动查询的成员为空，命令会报错并提示使用 `--users` 手动指定
> - 群主 / 管理员不在成员列表时会自动追加，不会报错
