## 常见踩坑

- **相对日期必须先跑 `date “+今天是 %Y年%m月%d日，星期%u”` 命令**：凭记忆推算”明天/下周四”会订错天，每次都先执行前置命令再构造参数
- **`--city` 是可选参数**：省略时跨所有城市搜索大厦；仅当建筑名在多城市有歧义时才需要填写
- **城市名不一定是城市**：部分城市归属大区（如南京 → “华东(苏沪)”），`--city` 找不到时先换大区名
- **建筑名太短会多重匹配**：`--building D` 可能匹配 D1/D2/D3，用 `--building 互联D2` 更具体
- **培训室超 4 小时需加 `--training`**：`--room-id` 模式必须手动加；搜索模式（`--building 培训`）自动识别，无需加
- **`--pick` 超出范围不降级**：结果不足 n 个时告知用户实际数量，让用户重新选择
- **设备/功能/配套说明**：
  - `--equips 电视机` → 筛选有电视机的会议室，结果 `🔧 设备` 行显示
  - `--equips Zoom` → 筛选支持 Zoom 的会议室，结果 `📹 功能` 行显示
  - `--equips 自然采光` → 筛选有自然采光的会议室，结果 `🌿 配套` 行显示
  - 多个条件同时传：`--equips 电视机 自然采光`（AND 关系，需同时满足）
  - 无设备/功能/配套时对应行不显示

## 故障排除

| 错误 | 原因 | 解决 |
|---|---|---|
| `401` / `未登录` | token 过期 | `--clear-cache` 后重试；或加 `--force-ciba` |
| `未找到城市/建筑` | 名称不匹配 | 换更具体的名称；部分城市在大区下，调整 `--city` |
| `时间参数不合法` | 非 5 分钟倍数 | CLI 会自动对齐，若仍报错检查日期格式 YYYY-MM-DD |
| `过去时间不可预订` | 日期推算出错 | 先跑 `date` 命令确认当前日期再重试 |
| `时长超出限制` | 普通会议室超 4 小时 | 拆分为两次预订；或换培训室加 `--training` |
| `同时预订超限` | 同一时刻已有 2 间 | 等其中一间结束后再订 |
| `监测任务创建失败(50801)` | 该时段已有监测任务 | 已有任务，大象有空时会通知，无需重复创建 |
| 进程挂起不退出 | SSE 连接未关闭 | 正常现象，命令执行完会自动退出；如卡住加 `--clear-cache` |

## 熔断机制

同一接口连续 5 次失败后熔断，直接返回错误不再重试（防触发风控）。重新运行命令即可恢复。`AUTH_EXPIRED` 不计入熔断次数，走降级认证流程。
