# pm-role-guide

**产品经理工作助手总入口** — 告诉我你在做什么，我帮你找到最合适的方法和工具。

## 使用方式

直接描述你的工作场景，例如：
- "负责搜索产品，DAU最近下跌了想分析原因"
- "要做Q2规划，不知道从哪里开始"
- "第一次写PRD，需要帮助"
- "有个功能需求，需要排优先级"

## 快速诊断：我现在最需要什么？

### 🚨 紧急情况
- 数据异常 → **`data-anomaly-detective`**
- 产品故障/舆情 → **`crisis-response-playbook`**
- 老板要求今天出方案 → **`strategy-doc-polisher`** + **`upward-management-kit`**

### 📊 每周例行
- 写周报 → **`data-insight-narrator`**
- 复盘上周 → **`retrospective-coach`**
- 检查产品健康度 → **`product-health-dashboard`**

### 🎯 季度规划
1. 先做目标体系 → **`north-star-metric-designer`** + **`kpi-tree-builder`**
2. 做 Roadmap → **`product-roadmap-writer`**
3. 做竞品分析 → **`competitive-landscape`**
4. OKR 质检 → **`okr-quality-checker`**

### 💡 产品设计
1. 理解用户真实需求 → **`jtbd-analyzer`**
2. 用户分层 → **`user-segmentation`**
3. 设计信息架构 → **`ia-designer`**
4. 验证转化路径 → **`conversion-optimizer`**

### 📈 增长问题
1. 诊断漏斗 → **`aarrr-funnel-analyzer`**
2. 分析留存 → **`retention-diagnosis`**
3. 设计增长循环 → **`growth-loop-designer`**
4. 跑实验 → **`ab-test-designer`** + **`growth-experiment-tracker`**

## 如果你没描述工作场景

我会先问你一个问题：你目前最想解决哪类问题？

- **A. 交付**（写 PRD、需求评审、配合研发上线）→ 路由到需求与交付
- **B. 数据增长**（指标分析、漏斗诊断、实验设计）→ 路由到数据与增长
- **C. 战略竞品**（产品方向、竞品分析、商业模式）→ 路由到战略与竞品
- **D. 协作汇报**（跨团队、向上汇报、OKR 规划）→ 路由到协作与汇报

如果场景横跨多个，优先路由到你**现在最迫切的那个任务**对应的 skill。
